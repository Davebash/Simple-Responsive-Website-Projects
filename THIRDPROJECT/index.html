Introduction

Java is a high-level, class-based, object-oriented programming language that is designed to have as few implementation dependencies as possible. It is a general-purpose programming language intended to let application developers write once, run anywhere (WORA), meaning that compiled Java code can run on all platforms that support Java without the need for recompilation.

Java applications are typically compiled to bytecode that can run on any Java Virtual Machine (JVM), regardless of the underlying computer architecture. The syntax of Java is similar to C and C++, but Java has fewer low-level facilities than either of them.

    Java Features

  • Object-Oriented: Java supports the principles of object-oriented programming such as inheritance, encapsulation, polymorphism, and abstraction.
  • Platform-Independent: Programs written in Java are compiled into bytecode, which can be run on any machine with a JVM
  • Automatic Memory Management: Java features automatic memory management and garbage collection, meaning developers do not need to manually manage memory allocation and deallocation.

Java vs. C++

Java and C++ have some significant differences:

  • Memory Management: Java uses an automatic garbage collector to manage memory, whereas C++ requires manual memory allocation and deallocation (using new, delete).
  • Platform Independence: Java is platform-independent due to the JVM. C++ code needs to be compiled separately for each platform.
  • Syntax and Complexity: Java is generally simpler with fewer low-level constructs like pointers, while C++ provides more fine-grained control over system resources.
  • Use Cases: Java is widely used in web applications, mobile development (Android), and enterprise software. C++ is used for game development, high- performance computing, and systems programming.
Object-Oriented Programming in Java

Java is a pure object-oriented language, where almost everything (except primitive types) is an object. Key OOP principles in Java include:

  • Encapsulation: Restricting direct access to an object's data and providing methods for interaction.
  • Inheritance: The ability for classes to inherit properties and methods from other classes.
  • Polymorphism: Allowing one interface to be used for different data types. Here's an example of a simple class in Java:
    • class Car { String brand; int speed; void accelerate() { speed += 10; } }

    The Car class encapsulates properties (brand, speed) and behavior (accelerate).

Hello World

Here is a simple "Hello, World!" example in Java:

public class Main { public static void main(String[] args) { System.out.println("Hello, World!"); } }

This code defines a Java class named Main with a main method, which serves as the entry point for any standalone Java application. The method prints "Hello, World!" to the console.

Variables

In both Java and C++, variables act as containers to store data values. However, Java variables fall into primitive and reference types. Primitive types include int, boolean, char, etc., while reference types store addresses of objects. In C++, variables are typically primitive, but pointers and references add complexity, allowing direct memory access. Java simplifies this by removing the need for pointers.

Declaring Variables

In Java, variables must be declared with a type:

int myNumber = 5; String name = "John";

C++ follows a similar structure but allows for more complexity, such as pointers:

int* ptr; // Pointer declaration in C++

Java's approach ensures more safety by abstracting memory management, while C++ offers more flexibility and control.

Variable Scope

Variable scope defines where in a program a variable can be accessed:

  • Local Scope: Variables declared inside a method (Java) or function (C++) are accessible only within that method or function.
  • Class Scope: Variables declared at the class level are accessible throughout the class. Java variables declared within a block {} have scope limited to that block, whereas C++ allows more control with pointers and references.
Global Variables

Global variables are variables accessible from any part of the program:

  • Java: Global variables are often simulated using static class members, as Java doesn't support true global variables.
    • class MyClass { static int globalVar = 100; }
  • C++: Global variables can be declared outside of all functions and classes, making them accessible everywhere in the program.
Constants

Constants are fixed values that cannot change during program execution. In Java, you define constants using the final keyword:

final int MAX_SPEED = 120;

In C++, constants are declared using const:

const int MAX_SPEED = 120;

Java constants are typically declared within a class, whereas C++ constants can be declared globally or locally.

Data Types

Java and C++ both support basic data types like int, float, and char. Java also provides wrapper classes (Integer, Float) to convert primitive data into objects. C++ supports more low-level data management through pointers, structures, and unions, allowing you to manipulate memory directly. Both languages enforce strict typing, but C++ allows more flexibility through templates and type casting.

if...else statement

The if...else statement controls conditional execution in both Java and C++. Their syntax is very similar:

if (condition) { // code } else { // code }

The statement checks the condition: if true, the first block of code executes; otherwise, the code in the else block runs. In both languages, these statements are used to make decisions and control program flow.

while statement

Both Java and C++ use the while loop for repeating code as long as a condition is true:

while (condition) { // loop body }

This loop checks the condition before each iteration. If the condition is false at the start, the loop will not execute at all. It's commonly used for continuous checks or processing data streams.

Function declarations

Functions (or methods in Java) are reusable blocks of code that perform a specific task:

  • Java: Functions are always defined within classes and are known as methods. For example:
    • public void myFunction() { // code }
  • C++: Functions can exist globally or inside classes. For example:
    • void myFunction() { // code }

    Java's methods must belong to classes, while C++ supports both free functions and methods within classes, giving it more flexibility.
